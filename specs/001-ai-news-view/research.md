# リサーチ: 生成AIニュース閲覧画面

## 決定事項

### 1. デザインアプローチ
- **決定**: モダンでスタイリッシュなデザインを実現するため、CSS Modules と CSS Variables を活用したテーマ管理を行う。
- **理由**: TailwindCSSが導入されていないため、標準的なCSS機能で柔軟かつ軽量なスタイルを実現する。グラデーションやシャドウ、アニメーションを効果的に使用する。
- **代替案**: TailwindCSSの導入（構成変更が必要なため却下）。

### 2. 読了時間計算ロジック
- **決定**: 日本語の平均読字速度（約600文字/分）または英語（200語/分）を基準に計算する。仕様書には「1分=200語換算」とあるため、これを遵守する。
- **実装**: `word-count` ライブラリまたは単純な空白区切りカウントを使用。

### 3. データ取得
- **決定**: Firestoreから記事データを取得する。
- **理由**: プロジェクトの依存関係に `@google-cloud/firestore` があるため。
- **代替案**: Mock API（開発初期はMockを使用し、後でFirestoreに差し替える）。
